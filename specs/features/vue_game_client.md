# 网页版游戏大厅及牛牛房间（全栈Vue化）需求文档

## 1. 背景与目标
原基于 Cocos Creator 的 H5 客户端由于打包流程繁琐、配置容易丢失（如Prefab绑定、Bundle设置）、且网络连接调试困难，导致部署频繁失败（静默黑屏）。
为彻底解决前端打包与黑盒问题，决定将游戏前端界面（大厅和牛牛房间）**全部用 Vue 重新实现**，集成到现有的 `web-ui` 项目中，实现真正的全栈化。
保留原有的图片素材，但采用纯 HTML/CSS/Vue 进行界面布局，使用原生的 WebSocket 连接后端 C++ 游戏服务器。

## 2. 核心功能需求

### 2.1 网络通信层 (WebSocket Manager)
- 需要一个全局的 WebSocket 管理器，负责与后端游戏服的实时长连接。
- 支持接收并解析后端返回的 `msgpack` 二进制数据。
- 支持发送 JSON 对象并自动序列化为 `msgpack`。
- 处理心跳包、断线重连逻辑。
- 依赖：`msgpack-lite` 或类似库进行序列化/反序列化，`js-base64` 进行编码。

### 2.2 网页版游戏大厅 (Game Lobby)
- 作为用户登录后台后的一个独立页面或组件。
- 包含游戏列表（抢庄牛牛、百人牛牛、掼蛋等），使用原有的图片图标。
- 用户点击游戏图标时：
  1. 调用后台 HTTP 接口 `/player/game/create`（带对应 `gameType`）。
  2. 获取 `wsAddress` 和 `venueId`。
  3. 调用 WebSocket Manager 建立连接，并发送进入场地消息 (`MsgEnterVenue`)。
  4. 成功后跳转到对应的游戏房间页面。

### 2.3 抢庄牛牛房间 (GrabNiuNiu Room)
- 纯 Vue 实现的牌桌界面。
- **界面元素**：
  - 自己和其他玩家的头像、昵称、金币。
  - 桌面中央的公共区域。
  - 手牌区域（5张牌）。
  - 操作面板：准备按钮、抢庄倍数按钮组、下注倍数按钮组。
- **状态流转**：
  - 等待开始 (WaitStart) -> 准备。
  - 发4张牌 (Deal4)。
  - 抢庄 (GrabBanker) -> 显示抢庄按钮。
  - 下注 (Betting) -> 显示下注按钮。
  - 发最后1张牌 (Deal1)。
  - 比牌 (Compare) -> 翻开所有牌，计算牛几。
  - 结算 (Settlement) -> 飘字显示输赢金币。
- 使用原有的扑克牌图片素材进行渲染（可通过 CSS Sprite 或单张图片组合）。

### 2.4 素材复用
- 扑克牌图片、头像框、背景图、按钮图等均从原 `assets/` 目录中提取，放入 `web-ui/src/assets/images/game` 下。
