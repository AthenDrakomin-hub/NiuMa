- **涉及文件**：
  - `server/Server/GrabNiuNiu/GrabNiuNiuRoom.cpp`
- **依赖**：无

  - [ ] 状态机增加 `Deal1` 阶段，在下注结束后触发。

#### Task-03：暗箱拦截器与 Hot-swap 算法 (Manipulation Core)
- **预计工时**：75分钟
- **涉及文件**：
  - `server/Server/GrabNiuNiu/GrabNiuNiuRoom.cpp`
  - `server/Server/GrabNiuNiu/NiuNiuLogic.cpp` (如有，或新增)
- **依赖**：Task-02
- **验收标准**：
  - [ ] 在 `Deal1` 前查询内存/Redis中的玩家干预权重（ForceWin/ForceLose等）。
  - [ ] 根据目标牌型遍历 `RemainingDeck`，找到合适的第 5 张牌并与预留的暗牌交换。
  - [ ] 确保替换后场上牌不重复。

## 2. 关键路径
Task-02 (发牌模式) -> Task-03 (换牌算法) -> Task-05 (上帝面板) -> Task-07 (搓牌动画)

## 3. 风险评估
- **Task-03 (换牌算法)**：逻辑最复杂，容易出现找不到解或者导致死循环。应对策略：必须设置 Fallback 逻辑，找不到完美解时降级。
- **Task-07 (搓牌动画)**：移动端交互可能不流畅。应对策略：优先使用纯 CSS clip-path 配合 touchmove，避免复杂的 3D WebGL。
