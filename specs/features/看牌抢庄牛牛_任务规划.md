# 看牌抢庄牛牛开发计划

## 1. 任务拆分

### 第1阶段：底层牌型与倍率引擎升级（C++ 服务端）
预计工时：90分钟

#### Task-01：修改 `NiuNiuRule.h` 和 `NiuNiuRule.cpp`
- **预计工时**：45分钟
- **涉及文件**：
  - `server/Poker/NiuNiuRule.h`
  - `server/Poker/NiuNiuRule.cpp`
- **依赖**：无
- **验收标准**：
  - [ ] 增加五小牛、银牛、炸弹牛、五花牛的判定算法
  - [ ] 确保这些新牌型的权重优先级正确（五小牛最大）
- **风险**：低

#### Task-02：调整倍率配置逻辑
- **预计工时**：45分钟
- **涉及文件**：
  - `server/Server/NiuNiu100/NiuNiu100Room.cpp` (兼容百人牛牛)
  - 新建 `GrabNiuNiuRoom.cpp` 的倍率映射
- **依赖**：Task-01
- **验收标准**：
  - [ ] 正确应用 1x 到 10x (五小牛) 的赔率
- **风险**：低

### 第2阶段：看牌抢庄核心状态机（C++ 服务端）
预计工时：150分钟

#### Task-03：新增 `GrabNiuNiuRoom` 控制类
- **预计工时**：120分钟
- **涉及文件**：
  - `server/Server/GrabNiuNiu/GrabNiuNiuRoom.h/cpp`
  - `server/Server/GrabNiuNiu/GrabNiuNiuAvatar.h/cpp`
- **依赖**：Task-01, Task-02
- **验收标准**：
  - [ ] 完整实现状态机流转：`Deal4` -> `GrabBanker` -> `Betting` -> `Deal1` -> `Compare` -> `Settlement`
  - [ ] 实现抢庄规则：最高倍数当庄、平局随机、不抢则金币最多当庄
  - [ ] 实现闲家下注限额限制规则
- **风险**：高 (状态机较长，多并发请求易导致时序问题)

#### Task-04：看牌抢庄协议定义
- **预计工时**：30分钟
- **涉及文件**：
  - `server/Server/GrabNiuNiu/GrabNiuNiuMessages.h/cpp`
- **依赖**：无
- **验收标准**：

## 2. 关键路径
- **关键路径**：Task-01 → Task-03 → Task-06 → Task-09
- **关键路径工时**：300分钟
- **总预估工时**：540分钟 (约 9 小时)

## 3. 风险评估与应对
- **中高风险**：抢庄逻辑状态机
  - **应对**：使用严谨的 `std::map` 或位掩码记录所有玩家的抢庄倍数状态，直到所有人都表态（或定时器结束）才流转至下一个 `Betting` 状态。
- **中风险**：牌库预设冲突
  - **应对**：后台下发的卡牌必须优先从全局洗好的 52 张牌堆中剔除，然后再随机给其他无控制玩家发牌。
