# 看牌抢庄牛牛技术方案

## 1. 需求背景与目标
在现有百人牛牛的基础上，增加更具博弈性、策略性的“看牌抢庄牛牛”（支持 2 人与 5 人局）玩法。同时，开发完善后台管理端（Java+Vue）的牌局控制接口，实现管理员可以指定、预设对局的发牌结果，左右牌局输赢。

## 2. 核心功能拆解

### 2.1 基础牌型与赔率引擎升级 (C++ 底层)
目前 `NiuNiuRule` 已包含没牛~牛牛、炸弹、五花牛等。需要增加/修改：
*   **五小牛 (WuXiaoNiu)**：5张牌均 < 5，总和 <= 10。赔率：**8倍** (最大牌型)。
*   **炸弹牛 (ZhaDan)**：4张牌点数相同。赔率提升至：**6倍**。
*   **五花牛/金牛 (WuHua)**：5张全为 J, Q, K。赔率：**5倍**。
*   **银牛 (YinNiu)**：5张全为 10, J, Q, K。赔率：**4倍**。
*   **牛牛**：**3倍**。
*   **牛七~牛九**：**2倍**。
*   **没牛~牛六**：**1倍**。

### 2.2 新玩法：看牌抢庄流程控制 (C++ 服务端 + TS 客户端)
创建全新的 `GrabNiuNiuRoom`（继承自基类或新开房间类型）。
**状态机流程 (State Machine)**：
1.  **WaitStart (等待开始)**：满员或点击开始。
2.  **Deal4 (发4张牌)**：给每位玩家发4张明牌（仅自己可见）。
3.  **GrabBanker (抢庄阶段)**：玩家根据4张牌选择抢庄倍数（不抢, 1x, 2x, 3x, 4x）。
    *   *定庄逻辑*：倍数最高者当庄；最高倍数相同则随机；都不抢则最高金币者当庄（5人局）或随机（2人局）。
4.  **Betting (下注阶段)**：闲家根据庄家倍数和自己金币余额选择下注倍数。
5.  **Deal1 (发第5张牌)**：系统下发第5张牌。
6.  **Compare (比牌阶段)**：闲家分别与庄家比大小。
7.  **Settlement (结算阶段)**：算分、抽水（1%），解散或继续下一局。

### 2.3 后台管理端发牌控制 (Java/Vue + C++)
**目标**：后台能够预设某局某人的第 5 张牌，或直接控制整副手牌。
*   **Vue 前端**：在对局管理/房间管理页面，新增【牌局干预】弹窗，可选中房间ID及座位号，下发卡牌数据。
*   **Java 后端**：新增 API `/api/admin/niuniu/preset`，验证权限后将数据写入 Redis 缓存（或通过内部 RPC/TCP 转发给 C++ 游戏服）。
*   **C++ 服务端**：在 `GrabNiuNiuRoom` 和 `NiuNiu100Room` 发牌阶段前，检查 Redis 中是否存在该局的“预设牌缓存”。若存在，则优先从牌库抽出预设卡牌发给对应玩家，剩余牌再随机下发。

## 3. 模块架构与数据流向
1. **[Vue Admin]** -> 发送 HTTP POST (预设卡牌) -> **[Java Admin Server]**
2. **[Java Admin Server]** -> 写入 Redis Key (`NiuMa:PresetCard:RoomID:RoundID`) -> 返回成功
3. **[C++ Game Server]** -> `DealCard` 状态时读取 Redis -> 覆盖洗牌结果 -> 发送 `MsgDealCard` 给 **[Cocos TS Client]**
4. **[Cocos TS Client]** -> 渲染 UI (4张暗牌 -> 抢庄UI -> 下注UI -> 第5张牌展示)。
